# ENVIRONMENT SETUP FOR RENT-A-CULTIVATOR

To fix the "Error Adding Machine" issue, you need to create a .env file in the project root with the following content:

## Create a .env file in the project root:

MONGODB_URI="your-mongodb-atlas-connection-string"
PORT=3000
ADMIN_NAME="Admin"
ADMIN_EMAIL="admin@example.com"
ADMIN_PASSWORD="Admin123!"
ADMIN_PHONE="0000000000"

## Steps to fix:

1. Create a .env file in the project root directory
2. Add your MongoDB connection string to MONGODB_URI
3. Restart the server with: npm start
4. Test the health endpoint: http://localhost:3000/health
5. Try adding a machine again

## MongoDB Connection String Format:
mongodb+srv://username:password@cluster.mongodb.net/database-name?retryWrites=true&w=majority

## Common Issues:
- Missing .env file
- Invalid MongoDB connection string
- MongoDB service not running
- Network/firewall blocking connection

## Test Endpoints:
- Health: http://localhost:3000/health
- Database Test: http://localhost:3000/test-db
- Machine Creation: POST http://localhost:3000/api/machines
